# 🌳 helpTree — Сервис взаимопомощи

[![Spring Boot](https://img.shields.io/badge/Spring%20Boot-3.4.3-brightgreen)](https://spring.io/projects/spring-boot)
[![Java](https://img.shields.io/badge/Java-21-orange)](https://www.oracle.com/java/)

[![Docker](https://img.shields.io/badge/Docker-24.0-2496ed)](https://www.docker.com/)

## О проекта

```powershell
**helpTree** — платформа для обмена помощью между пользователями.
 Проект позволяет создавать посты о помощи, откликаться на них и отслеживать рейтинг участников.
 Построен на микросервисной архитектуре с использованием современных технологий.
```
## Микросервисы

| Сервис | Порт | Назначение |
|--------|------|------------|
| **gateway-service** | 8080 | API Gateway (единая точка входа) |
| **helpTree-service** | 8081 | Пользователи, посты, помощь |
| **rating-service** | 8085 | Рейтинги и статистика |
| **notification-service** | 8087 | Telegram уведомления |

## Основные возможности
```powershell
- Создание и управление постами о помощи
- Система откликов (ACCEPTED → COMPLETED → CONFIRMED)
- Автоматический расчет рейтинга пользователей
- История изменений рейтинга
- Топ пользователей по рейтингу
- API Gateway как единая точка входа
- Асинхронное взаимодействие через Kafka
- Загрузка изображений в посты (MinIO/S3)
- Telegram уведомления о событиях помощи
- Система отзывов после завершения помощи
- Комментарии постов
- Чат между пользователями
- Система достижений (24 типа)
```

## Telegram уведомления

### Описание
Система уведомляет пользователей в Telegram о событиях связанных с помощью.

### Telegram Bot
- **Имя**: @helpTreeNotifierBot
- **Token**: `{TELEGRAM_BOT_TOKEN}` (переменная окружения)

### Команды бота
| Команда | Описание |
|---------|---------|
| `/start` | Показать Chat ID пользователя |
| `/help` | Список команд |
| `/myid` | Показать ваш Chat ID |

### Привязка Telegram к аккаунту
1. Открыть Telegram → написать `/start` боту
2. Скопировать Chat ID
3. В приложении: `PUT /api/users/{userId}/telegram` с телом `{"chatId": "ВАШ_CHAT_ID"}`

### События и уведомления

| Событие | Описание | Кому приходит |
|---------|----------|--------------|
| HELP_ACCEPTED | Кто-то принял помочь | **Автору** поста |
| HELP_COMPLETED | Помощник завершил работу | **Автору** поста |
| HELP_CONFIRMED | Автор подтвердил помощь | **Обоим** (автору и помощнику) |
| HELP_CANCELLED | Помощь отменена | **Обоим** (автору и помощнику) |



### Техническая реализация
- **Kafka Topic**: `help-events`
- **notification-service**: потребляет события из Kafka и отправляет в Telegram
- **HelpEvent DTO**: содержит postTitle, authorId, helperId, eventType

## Отзывы (Reviews)

### Описание
Система отзывов позволяет пользователям оставлять отзывы после завершения помощи. **Важно**: отзывы не влияют на рейтинг пользователя — рейтинг рассчитывается автоматически на основе событий помощи (HELP_CONFIRMED).

### Эндпоинты API

| Метод | URL | Описание | Авторизация |
|-------|-----|----------|-------------|
| POST | `/api/reviews` | Создать отзыв | Требуется |
| GET | `/api/reviews/help/{helpId}` | Получить отзывы по ID помощи | Публичный |
| GET | `/api/reviews/user/{userId}` | Получить отзывы о пользователе | Публичный |
| DELETE | `/api/reviews/{id}` | Удалить свой отзыв | Требуется |

### Создание отзыва

```json
POST http://localhost:8080/api/reviews
Content-Type: application/json
Authorization: Bearer {token}

{
    "helpId": 1,
    "rating": 5,
    "comment": "Отличная помощь! Рекомендую."
}
```


### Ответ

```json
{
    "id": 1,
    "helpId": 1,
    "fromUserId": 10,
    "fromUserName": "Иван",
    "toUserId": 5,
    "toUserName": "Пётр",
    "rating": 5,
    "comment": "Отличная помощь! Рекомендую.",
    "createdAt": "2026-03-23T14:30:00"
}
```

### Ограничения
- Отзыв можно оставить только на помощь в статусе `CONFIRMED`
- Один пользователь может оставить только один отзыв на одну помощь
- Автор помощи оставляет отзыв о помощнике (и наоборот)
- Автор помощи может удалить свой отзыв

### Техническая реализация
- **База данных**: таблица `reviews` (Flyway миграция V14)
- **Поля сущности**: helpId, fromUserId, toUserId, rating, comment, createdAt
- **Связи**: ManyToOne с Help, User (from), User (to)

## Комментарии к постам

### Описание
Система комментариев позволяет пользователям обсуждать посты о помощи. Поддерживаются **вложенные комментарии** (ответы на комментарии).

### Эндпоинты API

| Метод | URL | Описание | Авторизация |
|-------|-----|----------|-------------|
| POST | `/api/posts/{postId}/comments` | Создать комментарий | Требуется (X-User-Id) |
| GET | `/api/posts/{postId}/comments` | Получить комментарии поста | Публичный |
| DELETE | `/api/posts/{postId}/comments/{commentId}` | Удалить свой комментарий | Требуется |

### Создание комментария

```http
POST http://localhost:8081/api/posts/1/comments
Content-Type: application/json
X-User-Id: 2

{
    "content": "Комментарий к посту"
}
```

### Создание вложенного комментария (ответа)

```http
POST http://localhost:8081/api/posts/1/comments
Content-Type: application/json
X-User-Id: 2

{
    "content": "Ответ на комментарий",
    "parentCommentId": 5
}
```

### Ответ

```json
{
    "id": 6,
    "postId": 1,
    "userId": 2,
    "userName": "Пользователь",
    "content": "Ответ на комментарий",
    "parentCommentId": 5,
    "createdAt": "2026-03-23T18:00:00"
}
```

### Ограничения
- Комментарий не может быть пустым (max 1000 символов)
- Удалить комментарий может только его автор
- При ответе родительский комментарий должен принадлежать тому же посту

### Техническая реализация
- **База данных**: таблица `comments` (Flyway миграция V15, V16)
- **Поля сущности**: content, postId, userId, parentCommentId (nullable)
- **Связи**: ManyToOne с Post, User
- **Внешний ключ**: FK на родительский комментарий с каскадным удалением

## Чат между пользователями

### Описание
Система личных сообщений позволяет пользователям общаться в приватном чате 1-на-1.

### Эндпоинты API

| Метод | URL | Описание | Авторизация |
|-------|-----|----------|-------------|
| POST | `/api/chats` | Создать чат с пользователем | Требуется |
| GET | `/api/chats` | Список чатов текущего пользователя | Требуется |
| POST | `/api/chats/{chatId}/messages` | Отправить сообщение | Требуется |
| GET | `/api/chats/{chatId}/messages` | Получить сообщения (пагинация) | Требуется |
| POST | `/api/chats/{chatId}/read` | Отметить сообщения как прочитанные | Требуется |

### Создание чата

```http
POST http://localhost:8081/api/chats
Content-Type: application/json
Authorization: Bearer {token}

{
    "participantId": 5
}
```

### Ответ

```json
{
    "id": 1,
    "participantId": 5,
    "participantName": "Пётр",
    "lastMessage": "Привет! Как дела?",
    "lastMessageAt": "2026-03-24T10:00:00",
    "unreadCount": 2
}
```

### Отправка сообщения

```http
POST http://localhost:8081/api/chats/1/messages
Content-Type: application/json
Authorization: Bearer {token}

{
    "content": "Привет!"
}
```

### Получение сообщений

```http
GET http://localhost:8081/api/chats/1/messages?page=0&size=20
Authorization: Bearer {token}
```

### Ответ (с пагинацией)

```json
{
    "content": [
        {
            "id": 1,
            "chatId": 1,
            "senderId": 5,
            "senderName": "Пётр",
            "content": "Привет!",
            "createdAt": "2026-03-24T10:00:00",
            "isRead": true
        },
        {
            "id": 2,
            "chatId": 1,
            "senderId": 2,
            "senderName": "Иван",
            "content": "Привет, Пётр!",
            "createdAt": "2026-03-24T10:01:00",
            "isRead": false
        }
    ],
    "totalElements": 2,
    "totalPages": 1,
    "number": 0
}
```

### Ограничения
- Нельзя создать чат с самим собой
- Чат уникален для пары пользователей (при повторном создании возвращается существующий)
- Сообщение max 5000 символов
- Удалить чат может только участник
- Удалить сообщение может только автор

### Дополнительные эндпоинты

| Метод | URL | Описание |
|-------|-----|----------|
| DELETE | `/api/chats/{chatId}` | Удалить чат |
| DELETE | `/api/chats/{chatId}/messages/{messageId}` | Удалить сообщение |

### Техническая реализация
- **База данных**: таблицы `chats`, `messages` (Flyway миграция V17)
- **Сущности**: Chat, Message
- **Пагинация**: Spring Data Pageable

## Система достижений

### Описание
Система достижений мотивирует пользователей помогать другим. Достижения выдаются автоматически при выполнении определённых условий.

### Просмотр достижений
1. Откройте профиль пользователя
2. Перейдите на вкладку **"Достижения"**

### Эндпоинты API

| Метод | URL | Описание | Авторизация |
|-------|-----|----------|-------------|
| GET | `/api/users/{userId}/achievements` | Получить достижения пользователя | Публичный |

### Ответ

```json
[
    {
        "id": 1,
        "type": "FIRST_HELP",
        "emoji": "🤝",
        "name": "Первая помощь",
        "description": "Помог 1 раз",
        "rarity": "COMMON",
        "earnedAt": "2026-03-27T10:00:00"
    }
]
```

### Список достижений

#### По количеству помощи

| Достижение | Описание | Редкость |
|------------|----------|----------|
| 🌱 Новичок | Зарегистрировался | COMMON |
| 🤝 Первая помощь | Помог 1 раз | COMMON |
| ⭐ Волонтёр | Помог 5 раз | COMMON |
| 💪 Благотворитель | Помог 10 раз | UNCOMMON |
| 👑 Меценат | Помог 25 раз | RARE |
| 🏆 Легенда | Помог 50 раз | EPIC |
| 💎 Алмазная рука | Помог 100 раз | LEGENDARY |

#### По скорости

| Достижение | Описание | Редкость |
|------------|----------|----------|
| ⚡ Скороход | Помог в течение часа | COMMON |
| 🦉 Ночной сова | Помог ночью (22:00-06:00) | COMMON |
| 🎉 Выходной герой | Помог в выходные | COMMON |

#### По общению

| Достижение | Описание | Редкость |
|------------|----------|----------|
| 💬 Общительный | Написал 1 сообщение | COMMON |
| 🗣️ Болтун | Написал 50 сообщений | UNCOMMON |
| 📝 Автор | Создал 1 пост | COMMON |
| ✍️ Публицист | Создал 10 постов | UNCOMMON |

#### По долгам/рейтингу

| Достижение | Описание | Редкость |
|------------|----------|----------|
| ⚖️ Без долгов | Долг = 0 при 5+ помощах | UNCOMMON |
| ⚖️ Мастер баланса | Помог > чем получил | RARE |
| ⭐⭐⭐⭐⭐ Идеальный | Рейтинг 5.0 | LEGENDARY |

#### Праздничные

| Достижение | Описание | Редкость |
|------------|----------|----------|
| 🎄 Новогодний волшебник | Помощь в декабре | COMMON |
| 🎃 Хэллоуинский герой | Помощь 31 октября | COMMON |
| 🎁 День рождения | Помощь в свой день рождения | RARE |
| ❄️ Зимний помощник | Помог зимой | RARE |

### Редкость и цвета

| Редкость | Цвет |
|----------|------|
| COMMON | Серый (#9ca3af) |
| UNCOMMON | Зелёный (#22c55e) |
| RARE | Синий (#3b82f6) |
| EPIC | Фиолетовый (#a855f7) |
| LEGENDARY | Золотой (#f59e0b) |

### Техническая реализация
- **База данных**: таблица `achievements` (Flyway миграция V19)
- **Enum**: AchievementType (24 типа достижений)
- **Интеграция**: AchievementService вызывается из HelpService при подтверждении помощи
- **Frontend**: вкладка "Достижения" в профиле пользователя
- **Уведомления**: toast-уведомление при получении нового достижения
